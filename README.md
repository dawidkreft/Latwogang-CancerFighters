An analytics dashboard for the **Łatwogang × Cancer Fighters** charity stream  
(22–27 April 2026) — **3,199,026 donations · 217M PLN raised**.

🌐 **Live:** [dawidkreft.github.io/Latwogang-CancerFighters](https://dawidkreft.github.io/Latwogang-CancerFighters/)

## Running locally

### Requirements

- Python 3.7+
- A browser with internet access (Tailwind, ECharts and D3.js are loaded from CDN)

### Steps

**1. Fetch raw data** *(one-time, takes several hours — 3.2M records)*

```bash
python3 fetch_donations.py
# Output: latwogang_wplaty.json (~650 MB)
```

**2. Preprocess into dashboard data files** *(one-time, ~2–3 minutes)*

```bash
cd latwogang-dashboard
python3 preprocess.py
# Reads ../latwogang_wplaty.json → generates data/*.json (~5 MB total)
```

**3. Start the local server and open the dashboard**

```bash
python3 serve.py           # opens http://localhost:8080 automatically
python3 serve.py 9090      # optional: custom port
```

> **Why a server?** The dashboard loads data files via `fetch()`, which requires HTTP — it won't work when opened directly as a `file://` URL.

## How the data was fetched

siepomaga.pl is a React SPA — there is no public API documentation. The fetch script was built by reverse-engineering the Vite-bundled JS chunks served by the site.

Key findings:
- **API base:** `https://www.siepomaga.pl/api/v1/payments`
- **Fundraiser ID:** `LZSw1Ox` (resolved via `/api/donor/web/v2/permalinks/latwogang`)
- **Pagination:** cursor-based using `after_id` + `after_value` (`state_changed_at` of the second-to-last item per page)
- **Page size:** max 200 records per request
- **Auth:** no token required, but a browser `User-Agent` header is needed (plain urllib gets HTTP 403)
